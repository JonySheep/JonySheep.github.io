JavaScript操作数组方法总结

Posted on 2019-05-18

​ 数组是我们在开发过程中最常用的数据结构，利用数组我们可以进行多个数据的集中处理，例如数据排序、数据增删改查等操作，也可以使用数组来进行字符串处理。

​ 在JavaScript中数组对应的内置对象是Array，它的原型是Object对象。因此数组不属于JS中的基本数据类型，而是对象类型。本篇博客将涉及以下内容：

  1. 判断Array对象的方法
  2. Array对象相关操作API总结

Array对象相关操作API总结

插入

push(…items: T[])

  • 功能：从尾部插入元素
  • Param：一个或多个元素值
  • Return：插入后的数组长度值

unshift(…items: T[])

  • 从头部插入元素
  • Param：一个或多个元素值
  • Return：插入后的数组长度值

splice(start: number, 0, …items: T[])

  • 从某个位置插入元素
  • Param：
    • start：插入位置
    • …items：插入的一个或多个元素值
  • Return：[] （本来应该返回删除的元素列表，但这里没有进行删除，所以返回空数组）

插入时直接对堆中的数组对象进行操作，不会进行拷贝

删除

pop()

  • 从尾部删除一个元素
  • Param：无
  • Return：被删除的元素值

shift()

  • 从头部删除一个元素
  • Param：无
  • Return：被删除的元素值

splice(start: number, deleteCount: number, …items: T[])

  • 从某个位置删除一个或多个元素，并且可以在删除位置插入元素
  • Param：
    • start：被删除元素的位置
    • deleteCount：被删除元素的数量
    • …items：插入的一个或多个元素值
  • Return：被删除的元素列表

查询

indexOf(searchElement: T, fromIndex?: number)

  • 查询某个元素第一次出现的位置，若没有这个元素则返回-1
  • Param：
    • searchElement: 查询的元素值
    • fromIndex：从指定位置开始查询
  • Return：第一次出现的元素下标

lastIndexOf(searchElement: T, fromIndex?: number)

  • 查询某个元素最后一次出现的位置，若没有这个元素则返回-1
  • Param：
    • searchElement: 查询的元素值
    • fromIndex：从指定位置开始查询
  • Return：最后一次出现的元素下标

includes(searchElement: number, fromIndex?: number)

  • 查询数组中是否包含指定元素，若没有这个元素则返回false
  • ES6中的新方法！
  • Param：
    • searchElement: 查询的元素值
    • fromIndex：从指定位置开始查询
  • Return：true | false

find(predicate: (value: number, index: number, obj: Int8Array) => boolean, thisArg?: any)

  • 查询第一个满足回调函数中指定条件的元素，若没有这个元素则返回undefined
  • Param：
    • predicate: 指定条件的回调函数
    • thisArg：执行回调时用作this 的对象
  • Return：第一次满足条件的元素值

findIndex(predicate: (value: number, index: number, obj: Int8Array) => boolean, thisArg?: any)

  • 查询第一个满足回调函数中指定条件的元素的下标，若没有这个元素则返回-1
  • Param：
    • predicate: 指定条件的回调函数
    • thisArg：执行回调时用作this 的对象
  • Return：第一次满足条件的元素下标

every(callbackfn: (value: number, index: number, array: Int8Array) => boolean, thisArg?: any)

  • 查询是否数组中所有元素都满足回调函数中的条件
  • Param：
    • callbackfn: 指定条件的回调函数
    • thisArg：执行回调时用作this 的对象
  • Return：true | false

some(callbackfn: (value: number, index: number, array: Int8Array) => boolean, thisArg?: any)

  • 查询是否数组中存在一个元素满足回调函数中的条件
  • Param：
    • callbackfn: 指定条件的回调函数
    • thisArg：执行回调时用作this 的对象
  • Return：true | false

filter(callbackfn: (value: T, index: number, array: T[]) => any, thisArg?: any)

  • 查询数组中所有满足回调函数条件的元素
  • Param：
    • callbackfn: 指定条件的回调函数
    • thisArg：执行回调时用作this 的对象
  • Return：所有满足回调函数条件的元素

遍历

keys()

  • 获得数组索引值的迭代器，使用迭代器进行元素遍历，与Object.keys(arr)的区别是会包含空元素的索引
  • IE浏览器不兼容！
  • Param：无
  • Return：数组索引值的迭代器

values()

  • 获得数组元素值的迭代器，使用迭代器进行元素遍历
  • IE浏览器不兼容！
  • Param：无
  • Return：数组元素值的迭代器

entries()

  • 获得包含数组键/值对的迭代器，使用迭代器进行元素遍历
  • IE浏览器不兼容！
  • Param：无
  • Return：数组索引值和元素值的迭代器

forEach(callbackfn: (value: T, index: number, array: T[]) => void, thisArg?: any)

  • 遍历数组，在回调函数中对每一个数组元素进行操作
  • 不在原数组上进行操作，进行了拷贝
  • Param：
    • callbackfn: 指定条件的回调函数
    • thisArg：执行回调时用作this 的对象
  • Return：无

map(callbackfn: (value: T, index: number, array: T[]) => U, thisArg?: any)

  • 创建一个新数组，其结果是该数组中的每个元素都调用一个提供的回调函数后返回的结果。
  • 不在原数组上进行操作，进行了拷贝
  • Param：
    • callbackfn: 指定条件的回调函数
    • thisArg：执行回调时用作this 的对象
  • Return：执行操作后的新数组

flatMap (callback: (this: This, value: T, index: number, array: T[]) => U|ReadonlyArray,thisArg?: This)

  • 与深度为1的map方法基本相同！只是会对结果数组执行一次扁平化
  • IE浏览器不兼容！

reduce(callback(accumulator, currentValue[, index[, array]])[, initialValue])

  • 对数组中的每一个函数执行定义的reducer函数（回调函数），将其结果汇总为单个返回值
  • 从左至右进行处理
  • Param：
    • callback：reducer函数
      • accumulator：上一次累加的结果
      • currentValue：当前值
      • currentIndex：当前索引
    • initialValue：初始累加值，若不写则默认为第0个元素
  • Return：处理结果

reduceRight(callback(accumulator, currentValue[, index[, array]])[, initialValue])

  • 与reduce作用相同，只是处理顺序不同，reduceRight从右至左进行处理

排序

sort(compareFn?: (a: T, b: T) => number)

  • 在原地进行数组排序，并且可以指定比较函数。对数字数组和字符串数组都可进行排序。
  • Param：
    • compareFn：比较函数
      • 若返回的number<0，a、b位置不变
      • 若返回的number=0，a、b位置不变（因此这个排序是稳定的：冒泡排序）
      • 若返回的number>0，a、b交换位置
  • Return：this

反转

reverse()

  • 在原地进行数组反转
  • Param：无
  • Return：该数组的引用，指向原数组

拼接

concat(…items: (T | ConcatArray)[])

  • 拼接一个或多个数组/值，并返回一个新数组，浅复制！
  • Param：多个数组/值
  • Return：拼接后的数组

扁平化

flat(depth?: number)

  • 以指定的深度递归遍历数组，达到扁平化的目的，返回一个新数组
  • IE浏览器不兼容！可使用reduce和concat函数替代
  • Param：遍历深度，默认为1，若为Infinity则为展开任意深度
  • Return：扁平化后的新数组

部分替换

copyWithin(target: number, start: number, end?: number)

  • 浅复制数组的一部分到同一数组的指定位置，会改变这个数组对象，但长度不变
  • IE浏览器不兼容！
  • Param：
    • target：指定复制的目标位置
    • start：指定被复制子序列的开始位置
    • end：可选，指定被复制子序列的结束位置-1，默认为数组长度
  • Return：this

fill(value: T, start?: number, end?: number)

  • 在原地以指定元素替换某个区间的所有元素
  • IE浏览器不兼容！
  • Param：
    • target：用以替换的元素值
    • start：指定被替换子序列的开始位置，默认为0
    • end：可选，指定被替换子序列的结束位置-1，默认为数组长度
  • this

截取

slice(begin? number, end? number)

  • 截取数组中某个指定区间内的元素，并返回一个新数组。若不传入参数，则作用为对数组进行浅拷贝。
  • Param：
    • start：可选，指定开始位置，默认为0
    • end：可选，指定结束位置-1，默认为数组长度
  • Return：截取后的新数组

转化为字符串

toString()

  • 将数组转化为字符串，默认使用 , 进行连接
  • Return：字符串

toLocaleString()

  • 与toString作用相同，只是使用本地字符编码进行转换

join(separator?: string)

  • 以指定的分隔符连接数组中的各个元素
  • Param：指定分隔符
  • Return：各个元素拼接后的字符串

转化为数组

Array.from<T, U>(iterable: Iterable | ArrayLike, mapfn: (v: T, k: number) => U, thisArg?: any)

  • 将传入的数据转化为数组，可传入的数据类型有
    • String
    • Set
    • Map
    • ArrayLike（伪数组，如arguments）
  • ES6新特性！IE浏览器不兼容！
  • Param：
    • iterable：需转化的数据
    • mapfn：可选，对数组中各个元素进行处理的回调函数
    • thisArg：可选，执行回调时用作this 的对象
  • Return：转化后的数组
