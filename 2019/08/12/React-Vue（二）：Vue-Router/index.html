React&Vue（二）：Vue-Router

Posted on 2019-08-12 |

随着前端技术的发展，大部分web应用都采用了单页(SPA)的形式进行页面跳转。与多页web应用不同，SPA使得用户能够减少页面切换的开销，提升使用专注度与使用效率。而在实现机制上，浏览器引擎提供了History和Hash两种跳转模式。

Vue-Router作为封装在Vue框架内部的跳转模块，在源文件中以router.js的文件形式存在，使得用户能够以一种更直接而集中的方式，对SPA应用的路由进行管理。

History和Hash

Hash和History是浏览器引擎中提供的两种前端路由跳转方式，History是在HTML5中才提出的新模式。Vue-Router内部支持了两种模式以供用户灵活选择，下面我们2种方式进行一个对比：

Hash History
Url中有无锚点(#) 有 无
Url改变时是否重载 否 是
事件监听函数 hashchange popstate
相关api 无 直接通过修改location.hash实现跳转 go(n); back(); forward(); pushState(object)；

从功能上来说，history作为较新的跳转方式，api更完整，易用性更高，因此在大多数情况下，我们会选用history mode。然而hash模式也有它特定的选型背景，因此在实际使用时，需要灵活选用。

Vue-Router

route包含的对象属性

  • path：路由路径

  • name：路由名称

  • component：渲染组件

  • children：子路由列表

  • props：传递到组件中的参数,可以以函数返回值的形式定义 (route) => ({ query: route.query.q }) }

  • redirect：重定向路径

  • alias：路径别名，当/a的路径设置了/b的别名时，打开/b时将会渲染/a对应的页面

  • meta：路径元数据

  • beforeEach(to, from, next) 导航守卫函数，用来捕获每一次路径跳转，定义在跳转前进行的操作

html-tag

定义了一个路由视图，在这个路由视图内的只会在这个视图区域内进行切换跳转。因此除了在App.vue中使用外，还可以用来固定导航栏，自定义切换页面。

可以看作是的一种框架内部封装，点击后将直接跳转到指定页面。并且在:to属性中可以绑定变量，或使用正则表达式实现匹配跳转。

内部对象

$router：访问路由列表实例，通过console可以看出该实例包含了history对象，matcher对象以及options中的routes数组。并且实例中包含2个重要的跳转函数push和replace，2个函数的参数都是路由对象，二者的区别在于：push是将新的页面加入路由栈中，而replace是直接替换当前页面。

$route：访问当前页面的路由实例，其属性(path, name, params, query)与push时传入的对象相对应。
