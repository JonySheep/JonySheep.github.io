CSS权威指南笔记(二)

Posted on 2020-03-07 |

第二章 选择符

1. 元素选择符

`h1 { color: black }`

2. 群组选择符

* 群组选择符可以为多个元素设置相同的样式
`h1,p { color: black }`

3. 类选择符

`.title { color: black }`
`.title.prompt { color: black }`

4. ID选择符

`#input-name { color: yellow }`

5. 属性选择符

* 简单属性选择符
`[moons] {  color: yellow }` 
具有moons属性的pml-planet元素应用此样式
* 根据精准属性值选择
`a[href]="http://baidu.com"] { color: yellow }`
`p[class="urgent warning"] 等于 p.urgent.warning`
* 根据部分属性值选择
`*[lang|="en"]` 选择lang属性以”en-“开头的元素
`p[class~="warning"]` 选择class属性中包含”warning”的p元素，与`p.warning等效`
`a[href*=".com"]` 选择href属性中包含”.com”子串的a元素，注意和~区分！
`a[href^="https"]` 选择href以“https”开头的a元素
`a[href$=".com"]` 选择href以”.com”结尾的a元素

6. 后代选择符

`div p { color: yellow }`

7. 子元素选择符

`h1 > strong { color: yellow }`

8. 紧邻同胞选择符

`h1 + p { margin: 0 }`
* 这里选择的紧邻同胞必须是紧跟在h1后面，且只能选择一个元素！因此以下代码中的p无法匹配此样式

9. 同胞选择符

* 同胞选择符可以选择一个元素后的所有匹配元素，只要他们属于同一个父元素
`h1 ~ p { margin: 0 }`

10. 伪类选择符

* 为文档中不一定真实存在的结构指定样式，或为某些元素的特定状态设置 _幽灵类_
* ⚠️伪类是依附于当前元素上的
* 常用结构伪类：
    * :root 选择根元素
    * :empty 选择没有任何子代的元素，包括文本节点、空白和换行符都不允许存在
    * :only-child 选择是唯一子代的元素，该元素不能有同胞元素
    * :only-of-type 选择是唯一一种子代元素，例如`a[href] img:only-of-type`选择具有href属性的a元素子代中的唯一图像，但a元素可以有其他子元素
    * :first-child 选择第一个元素，常用于li
    * :last-child 选择最后一个元素
    * :nth-child() 常用的伪类！选择每第n个元素，如在列表中，使用`li:nth-child(2n)`选择偶数列表项，常用于装饰间隔样式
* 常用动态伪类：
    * :link 选择未访问过的链接
    * :visited 选择已访问的超链接，与:link互斥
    * :focus 选择当前获得输入焦点的元素
    * :hover 选择鼠标指针放置其上的元素
    * :active 选择由用户激活的元素
    * 推荐排列顺序：link-visited-focus-hover-active
* UI状态伪类
    * :require, :enabled, :disabled, :checked, :default, :valid, :invalid……

11. 伪元素选择符

* 原理是在文档中插入虚构的元素，用::来表示
* 常用伪元素：
    * ::first-letter 用于装饰元素首字母
    * ::first-line 用于装饰元素首行
    * ::before 用于插入前置元素
    * ::after 用于插入后置元素

第三章 特指度

1. 特指度

* 特指度可以理解为一组样式的优先级。样式加载时，用户代理会计算每个样式规则中选择符的特指度，将其依附到规则中的每个声明上，若多个属性声明有冲突，特指度高的胜出。
* 一个特指度值由四部分构成0，0，0，0，特指度规则如下：
    1. 若样式在行内声明，特指度加1，0，0，0
    2. 选择符中的每个ID属性值，特指度加0，1，0，0
    3. 选择符中的每个类属性值、属性选择、伪类，特指度加0，0，1，0
    4. 选择符中每个元素选择、伪元素，特指度加0，0，0，1
    5. 连结符和通用选择符不增加特指度
* 例子

* 特指度的比较可以参考二进制数的比较，如“1，0，0，0”高于“0，10，10，10”，左边的数优先比较
* ⚠️具有`!important`的声明为重要声明，其优先级高于所有不重要声明

2. 继承

* 某些声明的样式不仅应用到当前所指的元素上，也会应用到元素的后代上
* 关键点⚠️
    * 不是所有样式都会被继承，如border、以及盒模型的大多属性都不会被继承
    * 继承后的值没有特指度！所以若对子元素样式进行了直接声明，将会覆盖继承的样式

3. 层叠

* 层叠机制是CSS组合样式的机制，它结合了继承和特指度的规则，最终计算出元素的展示样式
* 规则
    1. 找到匹配特定元素的所有规则
    2. 按显式权重和来源排序：!important标记的权重最高，来源包括：创作人员、读者、用户代理，以下按权重从高到低列出：
        1. 读者提供的样式中以!important标记的声明
        2. 创作人员编写的样式中以!important标记的声明
        3. 创作人员编写的常规声明
        4. 读者提供的常规声明
        5. 用户代理的默认声明
    3. 按特指度排序
    4. 按前后位置排序：样式表中靠后的声明权重较高，导入的声明权重低于样式表声明

CSS权威指南笔记(一)

第一章 CSS基础

本章主要介绍了CSS的引用方法和CSS中的查询方法。首先，CSS（层叠样式表）依赖于HTML元素而存在，HTML元素有两种分类方式：

按内容

* 置换元素：文档的内容不直接表示元素内容，如img、video、input
* 非置换元素：元素内容由用户代理在元素自身生成的框中显示（书中是这样写的很绕，我的理解就是所见即所得），如span、div等几乎所有元素

按显示方式

* 块元素：默认生成一个填满父区域的框，且前后都断行，最常见的有p、div
* 行内元素：在一行文本内生成元素框，如a、strong
    > HTML中不允许行内元素嵌套块元素，但可以使用CSS来打破这种常规

1. 如何引入CSS：

  1. 外部样式表：link标签
    <link rel=“stylesheet” type=“text/css” href=“color.css” media=“all”>

    * 使用`rel=“alternate stylesheet”`创建候选样式表，用于为浏览器提供候选样式选择

  2. 文档样式表：style元素
    <style type=“text/css” > .... </style>

  3. @Import指令
    @import url(color.css) screen
  4. 行内样式
  5. HTTP链接：在HTTP首部引用css文件

2. 媒体查询

用于根据指定的媒体类型来选择特定样式

媒体类型
all，screen，print
使用场景
link元素的media属性
style元素的media属性
@import声明
@media声明
媒体描述符
由一个媒体类型和一个或多个特性组成
媒体特性描述符
常见的max-width、min-width、color、grid等等
